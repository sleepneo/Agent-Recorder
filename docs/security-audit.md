## 3. 认证与授权

### 3.1 API Key 认证

**实现位置**：`src/AgentRecorder.Infrastructure/ApiKeyAuth.cs`

**认证流程**：
1. 优先从环境变量 `AGENT_RECORDER_API_KEY` 读取密钥
2. 若环境变量为空，从本地文件 `<DataDir>/config/api-key.txt` 读取
3. 若文件不存在，自动生成 256 位随机密钥并保存到文件
4. 请求时验证 `X-Agent-Recorder-Key` 头

**测试结果**：

| 测试用例 | 结果 |
|----------|------|
| 无 API Key 访问敏感接口 | ✅ 返回 401 UNAUTHORIZED |
| 无效 API Key 访问 | ✅ 返回 403 FORBIDDEN |
| 有效 API Key 访问 | ✅ 正常访问 |
| 公共接口无需认证 | ✅ 正常访问 |

### 3.2 权限控制矩阵

| 接口 | 认证要求 | 说明 |
|------|----------|------|
| GET /capabilities | 否 | 公共信息 |
| GET /permissions | 否 | 公共信息 |
| GET /displays | 否 | 公共信息 |
| GET /windows | 否 | 公共信息 |
| GET /windows/active | 否 | 公共信息 |
| GET /audio/devices | 否 | 公共信息 |
| POST /recordings | 是 | 写操作 |
| GET /recordings | 是 | 敏感数据 |
| GET /recordings/{id} | 是 | 敏感数据 |
| POST /recordings/{id}/stop | 是 | 写操作 |
| GET /confirmations/{id} | 是 | 敏感数据 |

**实现位置**：`src/AgentRecorder.Security/PolicyEngine.cs`

## 8. 风险评估

### 8.1 风险矩阵

| 风险描述 | 等级 | 可能性 | 影响 | 缓解措施 |
|----------|------|--------|------|----------|
| 远程未授权访问 | 高 | 低 | 高 | 本地绑定 + API Key |

| 路径遍历攻击 | 中 | 低 | 中 | 路径规范化 |
| 日志信息泄露 | 低 | 低 | 低 | 不记录敏感数据 |

### 8.2 已消除的风险

| 风险 | 消除方式 |
|------|----------|
| 网络暴露 | 仅绑定 localhost |

| 恶意路径写入 | 路径黑名单检查 |
